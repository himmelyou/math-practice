## 二、已完成的实现（当前代码）

### 2.1 密码与存储

- **后端**（`server/server.js`）
  - 学员密码：`bcrypt` 存储（`BCRYPT_ROUNDS = 10`），注册、登录、修改密码、管理员创建用户均走 bcrypt。
  - 兼容：若用户记录里仍是明文密码，登录时校验后会自动升级为 bcrypt 并写回。

### 2.2 学员会话与鉴权

- **后端**
  - 登录/注册成功：签发 JWT，写入 HTTP-only Cookie `auth_token`（7 天），生产环境 `secure` + `sameSite: none`。
  - 登出：`POST /api/logout` 清除 `auth_token` Cookie。
  - 鉴权中间件：`requireStudentAuth`（校验 Cookie 中的 JWT）、`ensureOwnData`（校验 `req.user.username === req.params.username`）。
  - 已保护接口：`GET/PUT /api/user/:username`、`GET/POST /api/user/:username/runs`、`POST /api/user/:username/change-password`。
- **前端**（`index.html`）
  - 所有与后端交互的 `fetch` 使用 `credentials: 'include'`。
  - 注销时调用 `POST /api/logout`。
  - 收到 401 时统一调用 `handleAuthRequired()`，回到登录页。

### 2.3 管理员与学员接口分离

- **后端**
  - 学员查自己：`GET /api/user/:username`（需学员 JWT）。
  - 管理员查任意学员：`GET /api/admin/user/:username`（需 `X-Admin-Pin`），供 report 等后台页使用。
- **前端**
  - `report.html` 的 `loadUserInfo(username)` 已改为请求 `GET /api/admin/user/:username` 并附带 `X-Admin-Pin`（及 `credentials: 'include'`）。

### 2.4 管理员口令当前逻辑（后端）

- 口令来源优先级：`process.env.ADMIN_PIN` → `data/admin-pin.json` 的 `pin` → 默认 `"2026"`。
  - 即：**已支持环境变量与文件**，仅前端与策略待加固（见下节）。

## 三、待做 / 可选

### 3.1 管理员口令加固（建议下一步做）

- **目标**：口令不写死在前端、可配置、可修改，并可选增强策略。
- **后端现状**：已支持 `ADMIN_PIN` 环境变量与 `admin-pin.json`；`PUT /api/admin/pin`、`POST /api/admin/pin/reset` 已存在。
- **待定/可选**：
  1. **前端**  
     - `index.html` 内「难度配置」等仍带硬编码 `ADMIN_PIN`，应改为与 admin/report 一致：由管理员在页内输入一次口令，用 `sessionStorage` 保存并作为 `X-Admin-Pin` 使用。  
     - `report.html` 目前仍写死 `ADMIN_PIN = "2026"`，应改为同样「页内输入 + sessionStorage」或从 admin 入口带参/共享（若同源）。
  2. **策略**  
     - 口令复杂度：如最少 6 位、仅数字或允许字母数字（由你决定）。  
     - 修改/重置：保持「需旧口令才能改」；重置为默认仅在有旧口令时可用。  
  3. **防暴力破解（可选）**  
     - 对 `POST /api/login` 和/或管理员入口做按 IP 或按账号的限流（例如 5 次/分钟）。

### 3.2 其他可选

- **JWT_SECRET**：生产环境必须设置 `process.env.JWT_SECRET`，不要用代码中的默认 dev 值。
- **HTTPS**：部署平台开启 HTTPS；Cookie 已在生产环境使用 `secure: true`。

## 四、环境变量与部署备忘

| 变量 | 说明 | 生产建议 |
|------|------|----------|
| `JWT_SECRET` | 学员 JWT 签名密钥 | 必设，随机长字符串 |
| `ADMIN_PIN` | 管理员口令（优先于文件） | 可选，设则覆盖文件与默认值 |
| `NODE_ENV` / `RENDER` | 用于 Cookie `secure`、`sameSite` | 生产设为 production 或存在 RENDER |
| `DATA_DIR` | 数据目录 | 按部署持久化需要设置 |
| `PORT` | 服务端口 | 按平台要求 |
