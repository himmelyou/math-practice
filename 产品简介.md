# Jarvis Math Lab 计算游戏 · 产品简介

> 面向 AI、合作伙伴和未来开发者的产品说明，用于快速理解目标、功能与技术选型，可随时更新。

---

## 一、产品概述（做什么）

Jarvis Math Lab 是一款面向 K12 学生的**口算 / 心算训练产品**。  
当前形态为 H5 Web 应用（适配手机、平板、电脑），后端为 Node.js API 服务。  

产品通过分级题库、闯关/生存/训练等多种模式，帮助学生在有限时间内高频练习四则运算；  
同时为老师 / 家长 / 机构提供练习记录、错题与进步曲线等数据，以辅助教学决策。

---

## 二、核心使用流程（从学生视角）

1. **进入产品**
   - 学生在浏览器打开 H5 页面（后续将新增微信小程序入口）。
   - 可先使用「游客试玩」体验，正式使用需注册登录。

2. **登录与进入主页**
   - 当前版本：用户名 + 密码登录；游客模式有次数 / 功能限制。
   - 登录后进入「主页」，可看到：
     - 总积分、最近成绩摘要
     - 已解锁模式入口（生存、闯关、训练、错题本、统计）

3. **选择模式进行练习**
   - 选择任一模式开始做题（详见下节「游戏模式」）。
   - 每局结束后会看到本局成绩、错题情况，部分模式会影响难度等级。

4. **查看练习结果与进步**
   - 在个人统计页面查看：
     - 不同难度的错误率与平均用时
     - 按日/按难度的进步曲线
   - 在错题本中针对薄弱题型再次练习。

5. **老师 / 管理员查看报表**
   - 老师使用后台管理页面：
     - 管理学员账号
     - 查看单个学员的所有练习记录、错题、进步曲线
     - 导出数据或截图用于家校沟通

## 三、游戏模式（怎么练）

### 1. 闯关挑战（按级别通关）

- **定位**：主线模式，用于确定和调整学生当前「难度等级」。
- **难度结构**：共 16 级（L1～L16），从一位数加减到带括号四则运算，详见《难度等级说明》。
- **基本规则**：
  - 每关固定 10 题，题目来自当前难度等级。
  - 有倒计时（例如 60 秒）：答对加少量时间，答错不加时。
  - 根据本局正确率和用时判定：
    - 达到「升级条件」：提升一个等级。
    - 仅达到「及格条件」：保持原等级。
    - 未达「及格条件」：可能降级（最低不低于某保底等级）。
- **用途**：
  - 初次使用时快速估计学生当前水平；
  - 作为周期性测验观测难度进阶情况。

### 2. 训练模式（在合适难度上打磨）

- **定位**：在当前水平附近做「定点训练」，不追求快速升级。
- **进入条件**：通常需要先通过至少一局闯关，以获得初始等级。
- **基本规则**：
  - 每局固定题数（例如 15～20 题），从当前等级题库出题，可少量波动上下级。
  - 正向计时（从 0 开始累积），用于统计「平均每题用时」。
  - 根据本局正确率与用时综合评估：
    - 表现持续优秀：可能小幅升级；
    - 表现正常：保持等级；
    - 正确率较低：可能降级，以保证题目不过难。
- **用途**：
  - 让学生在「略有挑战但不挫败」的水平上反复训练。

### 3. 生存挑战（高压模式 + 排行榜）

- **定位**：偏游戏化的「闯关 + 排行榜」模式，用于激发动力与展示成果。
- **解锁条件**：需在闯关达到某一最低等级（例如 ≥ L9）后自动解锁。
- **基本规则**：
  - 初始有固定时间条（例如 30 秒），答对加时、答错减时或不加时。
  - 题目难度会随表现自动调整：
    - 连续答对（如 5 题）→ 难度升级；
    - 答错或错误率升高 → 难度下降。
  - 游戏在时间耗尽时结束，记录：
    - 存活总时长
    - 总得分（可综合用时、难度、正确率）
    - 错题数量、最高到达难度等。
- **排行榜**：
  - 按「通关用时 / 存活时长、错题数」等进行排序。
  - 展示前 50 名记录，并突出当前用户所在位置（若上榜）。
  - 每个用户通常仅保留一条最佳成绩进入排行榜。

### 4. 错题本（针对性巩固）

- **自动收集错题**：
  - 各模式中回答错误的题目会自动写入错题本（包含题干、正确答案、学生答案、难度等级等）。
- **练习方式**：
  - 在错题本页面，学生可以：
    - 按难度/题型筛选错题
    - 重新作答错题，系统记录是否再错
  - 可选策略：多次做对后将某错题标记为「已掌握」，不再频繁出现。

## 四、难度体系（练什么）

- **级别设计**：
  - 共 16 个难度等级（L1～L16），覆盖：
    - 一位数加减（入门）
    - 两位数加减（含进退位）
    - 乘法口诀
    - 两位乘一位与对应除法
    - 不带括号 / 带括号的混合四则运算
- **配置能力**：
  - 每个难度可配置：
    - 升级与及格的准确率阈值
    - 每局限时 / 建议用时
    - 单题出题规则（数值范围、是否必进位/必退位等）
- **详细出题逻辑**：
  - 见同目录下《难度等级说明》，该文件从 L1～L16 逐级描述：
    - 题型说明
    - 运算范围与约束
    - 示例题

## 五、数据与统计（怎么看效果）

- **个人层面**：
  - 汇总指标：
    - 总积分、总局数
    - 最佳生存时长、最高得分
    - 当前难度等级、最高达成等级
  - 近期记录：
    - 不同模式下最近若干局（如最近 10～20 局）的成绩列表。
- **难度分析**：
  - 按难度（L1～L16）统计：
    - 答题数
    - 错误率（错误题数 / 总题数）
    - 平均每题用时（秒）
  - 用于识别「某难度明显偏弱」的情况。
- **进步曲线**：
  - 基于「按题记录」构建时间序列：
    - 每日 / 每周的错误率变化
    - 每日 / 每周的平均用时变化
  - 图表呈现：折线图或柱状图，观察一段时间内的趋势。
- **排行榜**：
  - 生存挑战排行榜用于激励与展示，不影响教学难度调节。

## 六、用户与角色（谁在用）

- **学生 / 学员**
  - 直接使用 H5 或（未来）小程序做练习。
  - 关注点：是否有趣、难度是否合适、是否能看到自己进步。

  - 通过学生端或老师提供的数据了解孩子练习情况。
  - 关注点：是否有明显进步、是否针对薄弱点练习。

- **老师 / 机构**
  - 使用后台管理功能：
    - 批量创建学员账号（当前为后台口令 + 表单形式）
    - 查看单人 / 多人的练习数据与统计
  - 关注点：是否能支撑班级 / 机构层面的教与学闭环。

## 七、技术架构（怎么实现）

- **前端（H5 版本）**
  - 单页 HTML + CSS + JavaScript，无前端框架。
  - 使用 `fetch` 与后端 API 通信。
  - 使用 Chart.js 绘制前端统计图表。

- **后端**
  - Node.js + Express。
  - 认证：
    - 登录 / 注册后签发 JWT。
    - H5 版本目前使用 Bearer Token（本地存储）与 Cookie 双通道方案。
  - 持久化：
    - 使用 JSON 文件存储：
      - `users.json`：学员账号与简要统计。
      - `runs.json`：按局记录与按题明细。
      - `settings.json`：难度配置与升级参数。
    - 将来可以平滑迁移到关系型数据库或文档库。

- **部署**
  - H5 前端：部署在 GitHub Pages。
  - 后端 API：部署在 Render（或类似服务），通过 HTTPS 提供接口。
  - 配置信息：通过环境变量设置（如 `JWT_SECRET`、`ADMIN_PIN` 等）。
